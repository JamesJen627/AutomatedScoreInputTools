# Automated Score Input Tools（ASIT）

# 第八章 系统配置与参数管理（System Configuration & Parameter Management）

Version：V1.0

---

# 8.1 模块目标

系统配置模块负责管理所有可配置参数。

配置必须与业务逻辑分离。

评分引擎、Excel解析器、导出模块不得写死任何可配置内容。

所有配置统一由 Configuration Manager 管理。

---

# 8.2 设计原则

配置遵循以下原则：

- 所有配置集中管理
- 修改配置无需重新安装软件
- 修改配置无需修改程序代码
- 配置修改后立即生效（必要时提示重启）
- 所有配置支持恢复默认值
- 配置变更记录日志

包含：


| 配置项        | 默认值  | 可修改 |
| ---------- | ---- | --- |
| 软件语言       | 简体中文 | √   |
| 自动保存       | 开启   | √   |
| 自动保存间隔     | 30 秒 | √   |
| 最近文件数量     | 10   | √   |
| 启动时恢复上次工作  | 开启   | √   |
| 启动检查更新（预留） | 关闭   | √   |

# 8.6 评分配置

这是整个系统最重要的配置。

包括：


| 配置项        | 默认值     |
| ---------- | ------- |
| 当前评分标准     | 最新版本    |
| 自动审核       | 开启      |
| 计算完成自动审核   | 开启      |
| 总成绩保留小数    | 2 位     |
| 占比总和不为100% | Warning |
| 自动重新计算     | 开启      |


说明：

这些配置均不得影响评分算法本身，仅影响系统行为。

---

# 8.7 评分规则配置

为了适应不同学校需求，以下规则支持配置：


| 配置项              | 默认行为 |
| ---------------- | ---- |
| 是否允许未知评分标准       | 否    |
| 是否允许继续处理 Warning | 是    |
| 是否允许自动修复中文冒号     | 是    |
| 是否允许自动去除前后空格     | 是    |
| 是否允许自动转换百分比      | 是    |
| 是否允许自动恢复未保存数据    | 是    |


以上规则均需记录到配置文件。

# 8.12 配置对象

系统内部统一采用 Configuration 对象。

建议包含以下模块：

```

```

```
Configuration

├── General
├── Excel
├── Score
├── Export
├── Log
└── Advanced
```

所有模块通过统一接口读取配置。

禁止直接访问配置文件。

---

# 8.13 配置加载流程

系统启动：

```

```

```
启动

↓

读取 config.json

↓

校验配置

↓

修复缺失项

↓

生成 Configuration Object

↓

缓存

↓

供各模块读取
```

若配置文件不存在：

自动创建默认配置。

---

# 8.14 配置校验

每次读取配置时执行：

-  类型检查 
-  默认值补全 
-  非法值修正 
-  版本兼容检查 

例如：

```

```

```
decimalPlaces

=

-2
```

自动恢复：

```

```

```
2
```

并记录日志。

---

# 8.15 配置版本

配置文件包含版本号。

例如：

```

```

```
{
  "version": "1.0.0"
}
```

软件升级后：

自动执行配置迁移。

确保旧版本配置继续可用。

---

# 8.16 配置恢复

设置页面提供：

**恢复默认配置**

恢复内容：

-  软件设置 
-  导出设置 
-  日志设置 
-  Excel 设置 

不会删除：

-  日志 
-  评分标准 
-  最近文件 

---

# 8.17 配置变更日志

任何配置修改均记录：


| 字段  | 内容               |
| --- | ---------------- |
| 时间  | 修改时间             |
| 配置项 | 修改项              |
| 原值  | 修改前              |
| 新值  | 修改后              |
| 操作人 | 当前用户（V1.0 默认为本机） |


便于追溯配置变化。

# 8.18 企业级建议（推荐纳入 V1.0）

为了提升系统可维护性，建议增加以下配置能力：

### ① 配置导入/导出

支持：

-  导出配置（config.json） 
-  导入配置（config.json） 

方便学校在多台电脑快速统一软件设置。

---

### ② 配置模板

支持保存多个配置模板，例如：

-  默认配置 
-  初中体育配置 
-  高中体育配置 
-  学校A配置 

用户可一键切换。

---

### ③ 配置锁定（预留）

预留管理员锁定功能。

例如：

-  禁止修改评分标准 
-  禁止修改导出路径 
-  禁止修改评分规则 

V1.0 不启用，但架构应预留扩展点。
