# 第三章 成绩计算与评分规则（Scoring Engine Requirements）

Version：V1.1（2026-06-27 与当前实现对齐）

---

# 3.1 模块目标

评分引擎（Scoring Engine）负责完成整个系统所有成绩计算工作。

职责包括：

- 加载评分标准
- 解析评分标准
- 根据学生成绩查找对应分数
- 计算各项目得分
- 计算总成绩
- 校验计算结果
- 输出计算报告

评分引擎不得负责：

- Excel读取
- Excel编辑
- 页面显示
- 导出Excel

评分引擎必须保持纯业务逻辑（Pure Business Logic）。

# 3.7 区间匹配规则（Floor Rule）

系统统一采用区间匹配。

不得采用：

- 四舍五入
- 插值
- 平均值
- 最近值

例如：

评分表：


| 成绩  | 得分  |
| --- | --- |
| 100 | 100 |
| 90  | 95  |
| 80  | 90  |


学生：

95

结果：

95分

不是100分。

# 3.11 总成绩计算

公式固定：

总成绩

=

坐位体前屈得分 × 占比

- +

800米得分 × 占比

- +

50米得分 × 占比

- +

立定跳远得分 × 占比

- +

仰卧起坐得分 × 占比

结果：

保留两位小数。

采用：

四舍五入。

---

# 3.11-b 单项得分档位

单项得分必须命中官方《成绩单项评分表》规定的 **20 档整数**：

100, 95, 90, 85, 80, 78, 76, 74, 72, 70, 68, 66, 64, 62, 60, 50, 40, 30, 20, 10

不得插值或四舍五入为其他整数。总成绩可为小数（保留两位）。

缺考项单项得分为 **0**（不在上述档位集合中，作为特例允许）。

---

# 3.11-c 总成绩等级

以该生 **maxPossibleTotalScore**（= Σ 100 × 各项目占比）为满分基准：

| 等级 | 条件 |
|------|------|
| 优 | totalScore ≥ maxPossibleTotalScore × 90% |
| 良 | ≥ 80% 且 < 90% |
| 合 | ≥ 60% 且 < 80% |
| 不合格 | < 60% |

等级写入导出 Excel 最后一列；不参与总成绩公式。

# 3.12-b 年级匹配

评分时除性别外，还须根据**班级名称推断年级**（初一 / 初二 / 初三），选择对应评分标准。

推断规则（`resolveGradeLevelFromClassName`）：

| 班级示例 | 年级 |
| -------- | ---- |
| 701、七年级 | 初一 |
| 801、八年级 | 初二 |
| 901、九年级 | 初三 |
| 含「初一」「初二」「初三」字样 | 对应年级 |
| 无法识别 | 默认 **初二** |

查表键：`项目编码 + 性别 + 年级`。

若缺少对应规则 → 该生计算失败（`success = false`），各项得分置 0，并返回 `errorMessage`。

---

# 3.12-c 缺考（空成绩）

单项 performance 为空（NaN）时：

- 该项 `itemScore = 0`
- 该项 `contributionScore = 0`
- 整体计算仍可 `success = true`（前提：评分标准齐全）

# 3.21 评分引擎设计原则

评分引擎必须遵循以下原则：

1. 不依赖UI。
2. 不直接操作Excel。
3. 不保存配置。
4. 不读取界面状态。
5. 所有算法可单元测试。
6. 同一输入必须产生同一输出（幂等性）。
7. 算法模块支持未来新增评分项目，无需修改核心流程。
