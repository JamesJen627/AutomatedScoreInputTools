# Automated Score Input Tools（ASIT）

# 第六章 业务规则、异常处理与系统行为（Business Rules, Exception Handling & System Behavior）

**Version：V1.0**

---

# 6.1 模块目标

本章定义系统所有业务规则、异常处理策略及系统行为。

所有模块（导入、检查、评分、审核、导出）必须遵循本章规范。

任何未在本章定义的异常，不得静默忽略。

系统必须：

- 明确发现异常
- 明确记录异常
- 明确提示用户
- 保证系统继续运行（可恢复异常）
- 保证数据一致性

---

# 6.2 业务规则总览

整个系统遵循以下业务规则：

1. 原始 Excel 文件永远只读，不允许直接修改。
2. 所有编辑均基于内存副本。
3. 所有计算必须先通过数据校验。
4. 所有导出必须通过审核。
5. 同一输入数据必须产生同一输出（幂等性）。
6. 所有异常必须记录日志。
7. 所有用户修改均可撤销（Undo）。
8. 软件崩溃后应尽可能恢复最近编辑状态。
9. 所有计算结果必须可追溯。

---

# 6.3 数据校验失败处理

当数据校验失败时：

系统立即停止进入评分流程。

用户可：

- 在线修改
- 重新检查
- 放弃导入

不得进入：

- 成绩计算
- 审核
- 导出

# 6.19 数据一致性

系统必须保证：

-  原始成绩永远不修改。 
-  得分必须重新计算。 
-  总成绩必须由系统生成。 
-  导出结果必须与审核结果一致。 
-  所有日志均可追溯。

# 6.21 业务规则优先级

系统按以下顺序处理业务：

1.  文件合法性 
2.  表头检查 
3.  数据格式检查 
4.  数据完整性检查 
5.  数据唯一性检查 
6.  评分标准检查 
7.  预检查（Pre-flight Check） 
8.  成绩计算 
9.  自动审核 
10.  导出 

任何一步失败，停止后续流程。

## 我建议在第六章再增加两个重要模块

这一版我认为已经有 **97 分**，但还有两个值得加入的设计。

### ① 业务规则配置化（Rule Configuration）

目前有些规则（如成绩合理范围、是否允许占比总和不为 100%、是否允许未知评分标准继续计算）写在 PRD 中。

建议明确：

> **凡是可能因学校要求不同而变化的业务规则，都应尽量配置化，而不是写死在代码中。**

例如：


| 配置项         | 默认值    | 是否可修改 |
| ----------- | ------ | ----- |
| 占比总和必须为100% | 否（仅警告） | 是     |
| 未验证评分标准允许计算 | 是      | 是     |
| 自动保存间隔      | 30 秒   | 是     |
| 自动审核        | 开启     | 是     |


这样后续无需改代码即可适配不同学校。

---

### ② 全局业务约束（Global Constraints）

建议增加一节，统一规定：

-  不允许直接修改任何"得分"列，得分只能由评分引擎生成。 
-  不允许直接修改"总成绩"，总成绩只能由系统计算。 
-  修改任何原始成绩或占比后，相关得分和总成绩立即标记为失效。 
-  导出时必须保证"所有失效标记已清除"，否则禁止导出。 

这会让整个系统的数据流更加严谨，也避免用户误改计算结果。
