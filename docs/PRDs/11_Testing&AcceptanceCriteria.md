# 第十一章 测试计划与验收标准（Testing & Acceptance Criteria）

Version：V1.0

---

# 11.1 模块目标

本章定义系统各功能模块的测试要求、验收标准及质量目标。

所有功能开发完成后，必须通过本章规定的测试后，方可视为完成。

任何未通过测试的功能，不允许进入正式版本。

---

# 11.2 测试原则

系统测试遵循以下原则：

- 功能完整性测试
- 数据正确性测试
- 边界值测试
- 异常处理测试
- 性能测试
- 回归测试

所有测试均应记录结果。

---

# 11.3 测试分类

系统测试分为：

```text
单元测试（Unit Test）

↓

集成测试（Integration Test）

↓

系统测试（System Test）

↓

用户验收测试（UAT）

↓

发布测试（Release Test）

```

---

# 11.4 Excel 导入测试

测试内容：

- 正常 Excel 导入
- 空 Excel
- 损坏 Excel
- 缺少 Sheet
- 多 Sheet
- 非 Excel 文件
- 大文件导入

验收标准：

- 正确导入合法文件
- 正确提示非法文件
- 不发生程序崩溃

---

# 11.5 数据校验测试

测试内容：

- 缺少表头
- 表头拼写错误
- 空值
- 重复学号
- 重复考号
- 非法成绩
- 非法性别
- 非法占比
- 时间格式错误

验收标准：

所有错误均正确定位到：

- 行
- 列
- 单元格

错误提示明确。

---

# 11.6 评分引擎测试

必须覆盖：

## 正常成绩

能够正确计算得分。

---

## 边界值

例如：

成绩刚好等于评分标准。

成绩刚好高于评分标准。

成绩刚好低于评分标准。

必须符合：

Floor Rule（取下限）。

---

## 极限值

高于最高档。

低于最低档。

必须返回最高档或最低档得分。

---

## 不同性别

男女评分标准必须正确切换。

不得混用。

---

# 11.7 总成绩测试

验证：

每个项目：

得分 × 占比。

总成绩：

五项累加。

保留两位小数。

四舍五入。

与人工计算结果一致。

---

# 11.8 自动审核测试

系统重新计算：

第一次

↓

第二次

结果必须一致。

若存在差异：

审核失败。

禁止导出。

---

# 11.9 Excel 编辑器测试

必须验证：

- 编辑单元格
- 撤销
- 重做
- 复制
- 粘贴
- 查找
- 替换
- 自动保存
- 实时校验

所有功能正常。

---

# 11.10 导出测试

测试：

- 导出 Excel
- 文件重名
- 导出失败
- 权限不足
- 导出路径不存在

验收标准：

导出文件完整。

字段顺序正确。

原始文件不修改。

---

# 11.11 日志测试

验证：

以下操作均生成日志：

- 导入
- 编辑
- 检查
- 计算
- 审核
- 导出
- 修改配置
- 更换评分标准

日志内容完整。

---

# 11.12 性能测试

测试数据规模：


| 学生人数 | 最大允许耗时 |
| ---- | ------ |
| 100  | ≤0.5 秒 |
| 500  | ≤1 秒   |
| 1000 | ≤2 秒   |
| 5000 | ≤8 秒   |


计算过程中：

界面不得无响应。

---

# 11.13 稳定性测试

连续执行：

- 导入
- 编辑
- 计算
- 导出

100 次。

要求：

无崩溃。

无内存泄漏。

无数据丢失。

---

# 11.14 回归测试

每次修改：

评分算法

评分标准

Excel解析

导出模块

均必须重新执行完整测试。

确保旧功能不受影响。

---

# 11.15 用户验收标准（UAT）

系统满足以下条件时，可进入正式使用：

- 所有功能符合 PRD
- 无阻塞级 Bug
- 所有测试通过
- 所有错误提示正常
- 所有日志正常
- 导出结果正确
- 用户可独立完成完整流程

---

# 11.16 Bug 分级


| 等级  | 描述             | 是否允许发布 |
| --- | -------------- | ------ |
| P0  | 系统崩溃、数据错误、评分错误 | 否      |
| P1  | 核心功能异常         | 否      |
| P2  | 一般功能异常         | 视情况    |
| P3  | UI、提示文字等问题     | 可发布后修复 |


---

# 11.17 发布验收清单

发布前必须确认：

□ 所有测试通过

□ 所有模块完成

□ PRD 已同步更新

□ 无 P0 Bug

□ 无 P1 Bug

□ 评分结果人工抽检正确

□ 导出文件正确

□ 日志正常

□ 自动审核正常

□ 软件可正常安装运行

全部完成后，方可发布 V1.0。

---

# 11.18 自动化测试要求（推荐）

建议建立自动化测试体系。

至少覆盖：

- 数据校验
- 评分引擎
- 审核引擎
- 导出模块

每次代码提交自动执行测试。

任何测试失败，禁止合并代码。

---

# 11.19 测试数据集要求

项目应准备以下标准测试数据：

1. 正常数据集
2. 缺失表头数据集
3. 重复数据集
4. 边界值数据集
5. 极限值数据集
6. 非法格式数据集
7. 大数据量数据集（5000 条）

所有开发与测试均应使用统一数据集，保证测试结果可复现。

---

# 11.20 验收结论

系统只有在以下条件全部满足时，方可认定达到 V1.0 发布标准：

1. 所有功能实现完成。
2. 所有测试通过。
3. 所有业务规则符合 PRD。
4. 所有评分结果准确无误。
5. 系统运行稳定，无重大缺陷。
6. 用户能够独立完成导入、校验、计算、审核及导出全流程。



**11.21《测试覆盖矩阵（Requirement Traceability Matrix，RTM）》**。

也就是把 **PRD → 功能 → 测试用例** 建立一一对应关系，例如：


| PRD 编号 | 功能         | 测试编号          | 状态  |
| ------ | ---------- | ------------- | --- |
| 2.4    | Excel 检查   | TC-001~TC-015 | ✅   |
| 3.7    | Floor Rule | TC-020~TC-030 | ✅   |
| 3.11   | 总成绩计算      | TC-031~TC-040 | ✅   |
| 6.13   | 导出异常       | TC-070~TC-078 | ✅   |
