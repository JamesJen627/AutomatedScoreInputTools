# 第十二章 部署、发布与运维规范（Deployment, Release & Maintenance）

Version：V1.0

---

# 12.1 模块目标

本章定义软件的部署方式、发布规范、安装目录、数据存储、升级策略及后续维护要求。

确保系统具备可安装、可升级、可维护的交付能力。

---

# 12.2 支持平台

V1.0：

支持：

- Windows 10（64 位）
- Windows 11（64 位）

预留支持：

- macOS
- Ubuntu Linux
- Android

系统架构设计不得依赖 Windows 专有实现。

---

# 12.3 安装方式

支持两种安装方式：

### 安装版（Installer）

生成：

```text
AutomatedScoreInputTools_Setup.exe

```

特点：

- 自动创建快捷方式
- 自动注册卸载程序
- 自动创建用户数据目录

推荐普通用户使用。

---

### 绿色版（Portable）

生成：

```text
AutomatedScoreInputTools.exe

```

特点：

- 无需安装
- 可放入 U 盘运行
- 所有配置保存在程序目录

适用于学校机房、临时办公环境。

---

# 12.4 安装目录

默认安装路径：

```text
C:\Program Files\AutomatedScoreInputTools\

```

用户数据目录：

```text
%APPDATA%\AutomatedScoreInputTools\

```

包含：

```text
config/
logs/
cache/
autosave/
score-rules/

```

程序目录与用户数据目录必须分离。

---

# 12.5 首次启动

首次运行时：

系统自动：

- 创建配置文件
- 创建日志目录
- 创建缓存目录
- 创建自动保存目录
- 检查默认评分标准

若默认评分标准不存在：

进入评分标准导入向导。

---

# 12.6 自动升级（预留）

V1.0：

不启用自动升级。

预留升级接口。

未来支持：

- 检查更新
- 下载更新
- 增量升级
- 一键安装

升级不得影响：

- 用户配置
- 日志
- 自动保存数据
- 评分标准

---

# 12.7 数据迁移

软件升级后：

自动迁移：

- 配置
- 日志
- 缓存
- 评分标准

迁移失败：

恢复旧版本。

不得造成数据丢失。

---

# 12.8 自动备份

系统自动备份：

- 配置
- 自动保存数据

默认保留：

最近 10 个版本。

支持：

恢复历史版本。

---

# 12.9 日志管理

日志目录：

```text
logs/

```

日志按日期生成：

例如：

```text
2026-06-27.log

```

超过保留期限：

自动清理。

---

# 12.10 缓存管理

缓存包括：

- 已解析评分标准
- 最近打开文件
- 临时计算数据

缓存仅用于提高性能。

删除缓存不得影响用户正式数据。

---

# 12.11 卸载规范

卸载程序时：

默认删除：

- 程序文件

询问用户是否保留：

- 配置
- 日志
- 自动保存
- 评分标准

默认建议保留用户数据。

---

# 12.12 发布规范

每个正式版本必须包含：

- 安装程序（Setup）
- 绿色版（Portable）
- Release Notes（更新说明）
- 用户使用手册
- 默认评分标准（如适用）

所有发布文件应包含版本号。

---

# 12.13 版本命名规范

采用语义化版本（Semantic Versioning）：

格式：

```text
主版本号.次版本号.修订号

```

例如：

- 1.0.0
- 1.0.1
- 1.1.0
- 2.0.0

规则：

- 主版本：重大功能变更
- 次版本：新增功能
- 修订号：Bug 修复

---

# 12.14 发布流程

正式发布流程：

```text
需求确认

↓

开发完成

↓

代码评审

↓

测试通过

↓

生成安装包

↓

版本签名

↓

发布 Release

↓

归档文档

```

每一步均需完成后方可进入下一阶段。

---

# 12.15 运维要求

系统运行期间：

应保证：

- 自动保存正常
- 日志记录正常
- 配置可恢复
- 数据可追溯

出现异常时：

优先保证数据安全，其次保证功能可恢复。

---

# 12.16 文档交付

正式发布时，应同步交付：

- 产品需求文档（PRD）
- 用户操作手册
- 开发文档
- 更新日志
- 测试报告

确保后续维护有据可依。

---

# 12.17 发布验收

版本发布前必须确认：

□ 安装程序可正常安装

□ 绿色版可独立运行

□ 配置正常加载

□ 日志正常生成

□ 自动保存正常

□ 导入、校验、计算、审核、导出流程全部通过

□ 无阻塞级缺陷

满足以上条件后，方可发布正式版本。

---

# 12.18 运维设计原则

系统应遵循：

1. 数据优先。
2. 可恢复优先。
3. 配置独立。
4. 日志完整。
5. 发布可追溯。
6. 升级可回滚。
7. 用户数据永久优先于程序文件。
