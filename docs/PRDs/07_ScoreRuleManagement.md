# Automated Score Input Tools（ASIT）

# 第七章 评分标准管理（Score Rule Management）

**Version：V1.0**

---

# 7.1 模块目标

评分标准管理模块负责：

- 导入评分标准
- 验证评分标准
- 解析评分标准
- 管理多个评分标准
- 为评分引擎提供统一规则对象（Rule Object）

评分引擎不得直接读取 Excel。

必须通过评分标准管理模块获取评分规则。

---

# 7.2 设计原则

评分标准必须完全独立于程序代码。

任何年份、地区或学校的评分标准变更，都应通过更换评分标准文件完成，而不是修改程序。

系统核心代码不得写死任何评分数据。

---

# 7.3 支持的评分标准格式

V1.0 支持：

- `.xlsx`
- `.xls`

暂不支持：

- CSV
- JSON
- XML
- 数据库

后续版本可扩展。

---

# 7.4 评分标准生命周期

评分标准按照以下流程管理：

```

```

```
导入评分标准

↓

格式验证

↓

内容验证

↓

解析

↓

生成 Rule Object

↓

缓存

↓

供评分引擎使用

↓

卸载（切换评分标准）
```

---

# 7.5 评分标准基本信息

每个评分标准必须包含以下元数据：


| 字段   | 必填  | 示例              |
| ---- | --- | --------------- |
| 标准名称 | √   | 2025 国家学生体质健康标准 |
| 版本号  | √   | 2025            |
| 发布单位 | √   | 教育部             |
| 适用范围 | √   | 初中女生            |
| 创建日期 | √   | 2025-01-01      |
| 文件版本 | √   | 1.0             |


这些信息用于展示和日志记录。

---

# 7.6 评分标准结构要求

每个项目必须包含：

-  项目名称 
-  性别 
-  成绩值 
-  得分 

例如：


| 项目  | 性别  | 成绩   | 得分  |
| --- | --- | ---- | --- |
| 50米 | 女   | 7.50 | 100 |
| 50米 | 女   | 7.60 | 95  |


系统按照统一结构解析。

---

# 7.7 项目映射

系统内部使用统一项目编码。


| Excel 名称 | 系统编码         |
| -------- | ------------ |
| 坐位体前屈    | sitReach     |
| 800m     | run800       |
| 耐力跑      | run800       |
| 50米      | run50        |
| 立定跳远     | standingJump |
| 仰卧起坐     | sitUp        |


支持配置别名，但最终映射到统一编码。

---

# 7.8 性别规则

评分标准必须区分：

-  男 
-  女 

系统根据学生性别自动选择对应规则。

若某项目缺少对应性别评分：

禁止计算。

---

# 7.9 查表策略配置

每个项目必须声明查表策略。


| 项目    | 策略               |
| ----- | ---------------- |
| 坐位体前屈 | Higher Is Better |
| 立定跳远  | Higher Is Better |
| 仰卧起坐  | Higher Is Better |
| 50米   | Lower Is Better  |
| 800m  | Lower Is Better  |


评分引擎根据策略执行统一查表算法。

---

# 7.10 评分标准验证

导入评分标准时执行：

-  文件完整性检查 
-  必填字段检查 
-  项目完整性检查 
-  性别完整性检查 
-  成绩排序检查 
-  得分排序检查 
-  重复记录检查 

全部通过后才能启用。

---

# 7.11 当前评分标准

系统同时只能启用一个评分标准。

首页、状态栏和日志均显示当前评分标准名称及版本。

切换评分标准后：

-  清空当前计算结果 
-  标记数据需要重新计算 

---

# 7.12 缓存策略

评分标准解析完成后生成内存对象。

评分引擎直接读取内存对象。

不得在计算过程中重复读取 Excel。

切换评分标准时重新解析。

---

# 7.13 评分标准更新

支持：

-  导入新版本 
-  替换当前版本 
-  删除旧版本 

若新版本验证失败：

保持当前版本继续使用。

---

# 7.14 评分标准异常

以下情况禁止启用：

-  文件损坏 
-  缺少项目 
-  缺少性别 
-  成绩为空 
-  得分为空 
-  排序错误 
-  重复成绩 
-  非法得分 

系统生成详细错误报告，定位到具体工作表、行号和列号。

---

# 7.15 Rule Object（评分规则对象）

评分标准解析完成后，统一转换为 Rule Object。

评分引擎仅接收 Rule Object，不关心 Excel 文件结构。

这样未来支持 JSON、数据库或网络接口时，无需修改评分引擎。

---

# 7.16 多版本扩展（预留）

V1.0：

一次仅启用一个评分标准。

V2.0：

支持多个评分标准并存，例如：

-  2025 国家标准 
-  2026 国家标准 
-  福建省标准 
-  广东省标准 

用户可自由切换。

---

# 7.17 版本兼容

系统启动时检查评分标准版本。

若版本低于当前支持范围：

提示用户：

> 当前评分标准版本较旧，建议更新。

允许继续使用，但记录日志。

---

# 7.18 安全原则

评分标准属于核心业务资源。

系统不得自动修改评分标准文件。

所有编辑操作仅限查看和替换，不提供在线编辑评分标准功能（V1.0）。

---

# 我建议对第七章再做一个升级（强烈推荐）

我建议**把评分标准真正做成插件**，而不是普通 Excel。

也就是说，在软件目录中增加一个固定目录：

```

```

```
/ScoreRules/

├── 2025/
│   ├── rule.xlsx
│   ├── manifest.json
│   └── icon.png
│
├── 2026/
│   ├── rule.xlsx
│   ├── manifest.json
│   └── icon.png
```

其中 `manifest.json` 保存：

```

```

```
{
  "id": "cn.edu.2025",
  "name": "2025 国家学生体质健康标准",
  "version": "1.0.0",
  "author": "教育部",
  "supportGender": ["男", "女"],
  "supportedItems": [
    "sitReach",
    "run800",
    "run50",
    "standingJump",
    "sitUp"
  ]
}
```

这样以后：

-  更换评分标准只需要复制一个文件夹。 
-  支持不同地区、不同学校标准。 
-  不需要修改任何代码。 
-  Cursor 后续也能按照插件架构开发。
