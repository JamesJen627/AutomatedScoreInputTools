# 2.4 Excel检查模块

Excel检查必须在计算之前完成。

检查未通过：

禁止计算。

---

## 第一项

检查表头。

必须存在以下**逻辑字段**（可使用规范表头或 PRD §5.21 定义的**简写别名**）：

班级

考号

学号

姓名

性别

坐位体前屈成绩（单位：厘米） — 别名：`坐位体前屈成绩`

坐位体前屈得分占比 — 别名：`坐位体前屈占比`

800m成绩（单位：分·秒） — 别名：`800m成绩`

800m得分占比 — 别名：`800m占比`

50m成绩（单位：秒） — 别名：`50m成绩`

50m得分占比 — 别名：`50m占比`

立定跳远成绩（单位：米） — 别名：`立定跳远成绩`

立定跳远得分占比 — 别名：`立定跳远占比`

仰卧起坐成绩（单位：次） — 别名：`仰卧起坐成绩`

仰卧起坐得分占比 — 别名：`仰卧起坐占比`

缺少任意逻辑字段：

检查失败。

**额外说明**：

- 导入按**表头名称**映射列，不依赖固定列序号；中间「得分」列（如 `800m得分`）允许存在，导入时忽略。
- 编辑器内统一使用规范表头名称。

---

## 第二项

检查空值。

以下字段不能为空：

班级

考号

学号

姓名

性别

全部占比

**成绩字段（五项）允许为空**：

表示缺考或未测；校验阶段不报错；计算阶段该项得 **0 分**。

以下字段若为空则报错：

班级、考号、学号、姓名、性别、全部占比

发现空值（非成绩类）：

定位单元格。

允许用户修改。

---

## 第三项

检查重复数据。

检查：

学号

考号

发现重复：

标记红色。

禁止计算。

---

## 第四项

检查性别。

仅允许：

男

女

其它：

全部报错。

---

## 第五项

检查成绩格式。

坐位体前屈：

允许：

数字

允许负数

例如：

12.5

-2.1

不允许：

12cm

abc

空格

---

50米：

允许以下格式：

**小数秒**：7.56

**计时器写法**：8"7 表示 8.7 秒；7"56 表示 7.56 秒（秒与小数部分以 `"` 分隔）

不允许：

7秒

7s

仅含非法字符的混合格式

---

800米：

必须：

mm'ss''

例如：

3'25''



支持中英文引号自动转换。

例如：

3’25“

自动转换：

3'25"

其它格式：

全部报错。

---

立定跳远：

仅允许：

米

例如：

2.31

不得出现：

231cm

2米31

---

仰卧起坐：

仅允许整数。

例如：

46

不得出现：

46次

46.5

---

## 第六项

检查占比。

所有占比必须：

≥0

≤100

全部占比之和建议为100%。

若不是100%，弹出警告：

"当前占比总和不是100%，是否继续？"

允许继续，但系统记录日志。

---

## 第七项

检查非法字符。

例如：

@

￥

乱码

均视为非法。

---

## 第八项

检查评分标准是否存在。

若当前评分标准文件不存在：

禁止继续。

提示：

"请先导入评分标准。"

---

检查完成：

生成检查报告。

包括：

总人数

错误数量

警告数量

通过数量

## 2.4 Excel 检查模块（新增：预检查阶段）

在正式进入成绩计算之前，系统必须执行一次 **Pre-flight Check（预检查）**。

### 功能目标

确认当前运行环境、评分标准及数据状态均满足计算条件，避免计算过程中出现异常。

### 检查内容

系统依次检查：

#### （1）评分标准文件

检查：

- 当前是否已加载评分标准。
- 评分标准文件是否存在。
- 文件是否损坏。
- 文件是否可读取。

检查失败：

```

```

```
未检测到评分标准文件。

请先导入评分标准。
```

禁止进入计算。

---

#### （2）评分标准版本

检查：

评分标准是否符合系统支持格式。

例如：

```

```

```
评分标准版本：

2025

支持

√
```

若版本未知：

弹出提示：

```

```

```
当前评分标准未经验证。

是否继续？

【继续】

【取消】
```

系统记录日志。

---

#### （3）评分项目完整性

检查评分标准是否包含：

- 坐位体前屈 
- 800m 
- 50m 
- 立定跳远 
- 仰卧起坐

若缺少任意项目：

禁止计算。

---

#### （4）数据状态

检查：

是否还有：

- 未修复错误 
- 未保存修改 
- 重复数据 
- 非法数据

存在任何错误：

禁止进入计算。

---

#### （5）警告项

例如：

占比总和 ≠100%

此类属于警告。

允许继续。

弹出：

```

```

```
当前存在1项警告。

继续计算可能影响结果。

【继续】

【返回修改】
```

---

#### （6）Excel状态

检查：

是否已保存。

若未保存：

提示：

```

```

```
当前数据尚未保存。

是否保存？

【保存】

【不保存】

【取消】
```

---

#### （7）检查报告

全部完成后生成：

```

```

```
Pre-flight Check

评分标准

√

Excel数据

√

数据格式

√

错误

0

警告

1

状态

允许开始计算
```

只有全部检查通过，"开始计算"按钮才允许点击。

# 2.8 新增：统一提示规范（Message Specification）

整个软件所有提示采用统一风格。


| 类型      | 图标  | 是否阻塞 | 示例         |
| ------- | --- | ---- | ---------- |
| Success | ✅   | 否    | 导入成功       |
| Info    | ℹ   | 否    | 已自动保存      |
| Warning | ⚠   | 可继续  | 占比总和不是100% |
| Error   | ❌   | 是    | 成绩格式错误     |


所有提示必须包含：

- 标题 
- 详细原因 
- 建议解决方案 
- 操作按钮

禁止出现系统异常栈、英文报错或无法理解的技术信息。  

2.9新增**数据变更历史（History / Undo Log）**  

因为教师在修改 Excel 时，经常会出现：

- 改错了，不知道改了什么。
- 谁改了哪一项（未来多用户版本）。
- 想撤销到某一次修改。

因此建议系统记录每一次编辑，例如：


| 时间       | 行   | 字段  | 修改前  | 修改后  |
| -------- | --- | --- | ---- | ---- |
| 10:21:33 | 18  | 50米 | 7.85 | 7.58 |
| 10:22:05 | 36  | 姓名  | 空    | 张三   |


这样不仅支持无限撤销，也为未来多用户协作、审计和问题追溯打下基础，而且几乎不需要重新设计架构。
