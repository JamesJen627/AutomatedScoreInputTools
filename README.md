# Automated Score Input Tools（ASIT）

**体育成绩自动评分系统** — 面向学校体育成绩管理的 Windows 桌面应用。

> **语言说明**：当前版本界面与提示信息**仅支持中文**。

---

## 功能简介

ASIT 将 Excel 成绩表导入后，按《国家学生体质健康标准》单项评分表自动计算各项得分与总成绩，并在导出前完成数据校验与二次计算审核。

```
导入 Excel → 自动检查 → 在线编辑 → 开始计算 → 审核 → 导出 Excel
```

主要能力：

- **Excel 导入**：按表头名称映射列，支持简写表头；中间「得分」列自动忽略
- **数据校验**：检查表头、必填项、格式、重复学号/考号等；缺考项成绩可为空
- **在线编辑**：内置表格编辑器，修改后自动重新校验
- **自动评分**：五项体质测试（体前屈、800m、50m、立定跳远、仰卧起坐）按性别与年级查表
- **自动审核**：对同一批数据执行两次独立计算，结果必须一致才允许导出
- **导出结果**：22 列完整表（含各项得分、总成绩、等级）

## 使用流程

1. **导入成绩**  
   选择 `.xlsx` / `.xls` 成绩文件。系统按表头映射数据并在编辑器中展示。

2. **检查与编辑**  
   修复校验错误（红色标记）。检查通过后方可计算。

3. **加载评分标准**  
   首次运行会自动从内置 `ScoreRules/` 复制插件到用户目录并激活。可在「评分标准」页查看当前标准。

4. **开始计算**  
   确认预检报告后执行计算，可查看每位学生的计算轨迹。

5. **审核与导出**  
   审核通过后点击「导出 Excel」，默认文件名为 `原文件名_已计算.xlsx`。

## Excel 格式说明

### 输入（计算前必填逻辑字段）

| 字段 | 说明 |
|------|------|
| 班级、考号、学号、姓名、性别 | 必填，不可为空 |
| 五项成绩 | 允许为空（缺考计 0 分） |
| 五项得分占比 | 必填；建议合计 100 |

**规范表头**（编辑器内默认使用）：

```
班级、考号、学号、姓名、性别、
坐位体前屈成绩（单位：厘米）、坐位体前屈得分占比、
800m成绩（单位：分·秒）、800m得分占比、
50m成绩（单位：秒）、50m得分占比、
立定跳远成绩（单位：米）、立定跳远得分占比、
仰卧起坐成绩（单位：次）、仰卧起坐得分占比
```

**简写表头**（导入时自动识别）：如 `800m成绩`、`800m占比`、`坐位体前屈成绩` 等。完整别名见 `src/shared/constants/field-mapping.ts`。

### 成绩格式

| 项目 | 格式示例 |
|------|----------|
| 坐位体前屈 | 数字（厘米），可为负，如 `19`、`-2.1` |
| 800m / 耐力跑 | 分·秒，如 `3'30"` |
| 50m | 小数秒 `7.56`，或计时写法 `8"7`（= 8.7 秒） |
| 立定跳远 | 米，如 `1.71` |
| 仰卧起坐 | 整数（次），如 `58` |

### 导出（22 列）

在输入列基础上，按 **成绩 → 得分 → 占比** 交错插入系统计算的得分列，末尾追加 **总成绩** 与 **等级（优≥90%/良≥80%/合≥60%）**。

等级以该生最高可获总成绩（= Σ 100 × 各项目占比）为满分基准判定。

## 评分规则

- 内置标准：**2025 国家学生体质健康标准**（`ScoreRules/2025/`）
- 查表维度：**项目 × 性别 × 年级**（初一 / 初二 / 初三）
- 年级推断：由班级名称推断，如 `701` → 初一（首位数字 7/8/9 对应初一/初二/初三）
- 查表策略：成绩落在两档之间时，取**下限**对应得分（Floor Rule）

### 从官方表重新生成规则

将官方文件 `docs/成绩单项评分表.xlsx` 放入项目后执行：

```bash
npm run import-rules
```

会更新 `ScoreRules/2025/rule.xlsx`。应用启动时会自动同步较新版本到用户数据目录。

## 项目结构

```
AutomatedScoreInputTools/
├── electron/           # Electron 主进程与 preload
├── src/
│   ├── renderer/       # React 界面
│   ├── app/            # 应用层服务（导入、计算、导出）
│   ├── domain/         # 领域逻辑（校验、评分、审核、规则）
│   ├── infrastructure/ # Excel 解析、导出、存储
│   └── shared/         # 类型、常量、工具
├── ScoreRules/         # 内置评分标准插件
├── docs/PRDs/          # 产品需求文档
└── docs/测试.xlsx      # 示例测试数据
```

架构分层：**UI → Application → Domain → Infrastructure**。业务规则来自 Excel 规则文件，不在代码中硬编码分值表。
